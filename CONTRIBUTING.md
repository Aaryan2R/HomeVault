# Contributing to HomeVault

Thanks for your interest in contributing! HomeVault is open source and welcomes contributions of all kinds.

## Getting started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/HomeVault.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes and test them with `python app.py`
5. Push and open a pull request

## What to work on

Check the [Issues](https://github.com/Aaryan2R/HomeVault/issues) tab for open tasks. Issues labelled `good first issue` are beginner-friendly.

Current good first issues:
- Fix mobile UI edge cases
- Add file type icons for more formats
- Improve error pages (404, 403, 500)
- Add pagination for large file collections
- Write basic tests

## Pull request guidelines

- Keep PRs focused on one thing — smaller PRs are easier to review and faster to merge
- Describe what you changed and why in the PR description
- Test your changes before submitting
- If fixing a bug, explain how to reproduce the original bug

## Reporting bugs

Open an issue with:
- What you were doing
- What you expected to happen
- What actually happened
- Your Windows version and Python version

## Feature requests

Open an issue with the `enhancement` label describing what you want and why it would be useful.

## Questions

Open an issue with the `question` label or start a Discussion.

Thank you for helping make HomeVault better.
