HomeVault Viva Sheet

1. What is my project?

My project name is HomeVault.

HomeVault is a local private cloud storage web application.
It lets users upload, store, search, filter, download, and delete files on their own computer through a browser.

The main idea is to give a Google Drive or Google Photos like experience, but in a simple local system for home use.

It is useful for families who want to store photos, videos, and documents without depending fully on paid cloud storage.


2. What problem does it solve?

Many families run out of free cloud storage very quickly.
Buying more cloud storage every month can be expensive.
Keeping files manually on USB or SSD is also inconvenient.

So this project solves that by creating a local storage website on a home computer.
That way files can be organized and accessed more easily.


3. What technologies did I use?

I used:
- Python Flask for the backend
- SQLite for the database
- HTML for page structure
- CSS for design
- Jinja for dynamic templates
- Pillow for image thumbnail generation

Simple explanation:
- Flask handles requests
- SQLite stores file information
- HTML/CSS shows the interface
- Pillow makes small preview images for photos


4. How does the project work in simple words?

The project has two main parts:

1. Actual file storage
- uploaded files are saved in folders on the computer

2. Metadata storage
- file information is stored in the database

So the app does not save the whole file inside the database.
It only saves information about the file, such as:
- file name
- type
- size
- folder
- upload date

This makes the system simpler and faster.


5. What happens when a user uploads a file?

Step by step:

1. User selects a file in the browser
2. The browser sends it to the Flask server
3. Flask checks the file extension
4. Flask decides the category:
- Photos
- Videos
- Documents
- Others
5. Flask creates a unique name using UUID
6. The file is saved in the correct folder
7. File details are saved in SQLite
8. If it is a photo, a thumbnail is generated
9. The user is redirected back to the homepage


6. Why do I use UUID for file names?

UUID gives every uploaded file a unique name.

Example:
- original file name may be photo.jpg
- saved file name may become something like 6d7198ab-41fc-4a4d-a17b-fd2b203c0215.jpg

Why this is useful:
- avoids name conflicts
- two files with same name can still be stored safely
- makes storage management easier

The original filename is still kept in the database, so the user sees the real name.


7. What is stored in the database?

The database stores metadata, not the real file.

Main fields are:
- id
- filename
- original_name
- file_type
- file_size
- folder
- upload_date

Simple meaning:
- id = unique number
- filename = saved name on disk
- original_name = user's original file name
- file_type = file category
- file_size = size in bytes
- folder = storage folder
- upload_date = upload time


8. Why do I store metadata separately?

Because it is more efficient.

If I store only metadata in the database:
- listing files becomes fast
- searching becomes easier
- filtering becomes easier
- file handling stays simple

The actual file remains in the local storage folder.

This is a better design than storing all files directly in the database.


9. What does the homepage do?

The homepage shows:
- upload section
- search section
- category filter buttons
- file list table

Depending on what the user selects:
- if no filter is used, it shows all files
- if search is used, it shows matching files
- if a folder is selected, it shows only that folder's files


10. How does search work?

Search uses the file's original name.

Step by step:
1. user types text in search box
2. browser sends GET request with search value in URL
3. Flask reads the search value
4. Flask asks the database for matching filenames
5. matching files are shown on the page


11. How does folder filtering work?

Files are already grouped into:
- Photos
- Videos
- Documents
- Others

When user clicks one category button:
1. browser sends folder name in URL
2. Flask reads it
3. Flask gets matching files from the database
4. only those files are shown


12. How does download work?

When the user clicks download:
1. browser sends request with file ID
2. Flask checks the database
3. Flask finds the file path
4. Flask sends the file back to the browser

So download is based on the database record.


13. How does delete work?

When the user clicks delete:
1. Flask finds the file in the database
2. removes the database record
3. removes the real file from storage
4. removes the thumbnail if it exists
5. sends user back to homepage

So delete removes both:
- metadata
- actual stored file


14. What is thumbnail generation?

Thumbnail means a small preview image.

In this project:
- photo files get a thumbnail
- the thumbnail is stored in static/thumbnails

Why it is useful:
- makes the interface better
- user can quickly recognize images
- improves user experience

Thumbnails are created using the Pillow library.


15. Why are thumbnails named by file ID?

Because file ID is always unique.

Example:
- file ID = 6
- thumbnail name = 6.jpg

This avoids confusion and makes it easy to connect a file record with its preview image.


16. What are the main files in the project?

app.py
- main Flask app
- handles routes
- controls upload, download, delete, search, filter

database.py
- handles database operations
- creates table
- saves, gets, searches, deletes file records

thumbnailer.py
- generates image thumbnails

templates/index.html
- user interface page

static/css/style.css
- design and styling


17. What does each file do in easy words?

app.py = brain
It decides what happens when a user clicks something.

database.py = record keeper
It stores and retrieves file information.

thumbnailer.py = preview maker
It creates small photo preview images.

index.html = face of the project
It is what the user sees in the browser.

style.css = looks and design
It makes the page clean and presentable.


18. What is Flask?

Flask is a Python web framework.

Simple meaning:
It helps us make web applications using Python.

In this project Flask is used to:
- create routes
- receive uploaded files
- send HTML pages
- send downloadable files


19. What is SQLite?

SQLite is a lightweight database.

Simple meaning:
It stores data in a single file and does not need a separate server.

Why I used it:
- easy to set up
- good for student projects
- enough for metadata storage
- simple to explain


20. What is Jinja?

Jinja is a template engine used with Flask.

It lets Python send values into HTML.

Example:
- Python sends files
- HTML shows those files using Jinja syntax

So it helps make dynamic web pages.

24. What is the overall flow of the project?

Simple full flow:

Browser -> Flask -> Database/Storage -> Flask -> Browser

More clearly:
1. user performs action in browser
2. Flask receives request
3. Flask reads or writes data
4. SQLite stores metadata
5. storage folder stores actual files
6. Flask sends updated result back to browser
