Issue-by-issue status after the latest pass:

1. Launcher opens only homevault.local
- Status: fixed
- File: start_homevault.py

2. Launcher asks for admin every time
- Status: improved

3. Launcher depends on C:\nginx
- Status: accepted for submission
- File: start_homevault.py

4. .bat launcher uses relative paths only
- Status: fixed
- File: HomeVault.bat
- The batch file now switches to its own folder first with cd /d "%~dp0".

5. Requirements install runs on every launcher start
- Status: fixed
- File: start_homevault.py
- Launcher now uses a simple .requirements_ready flag file so it does not fully rerun installs every time.

6. mDNS is working on the PC side but not guaranteed on every device
- Status: accepted for submission
- File: mdns_broadcast.py / network environment
- Local broadcasting works, but cross-device .local access still depends on router, phone browser, and discovery behavior.
- Safe fallback remains the direct LAN IP.

7. Encoding garbage in some templates/files
- Status: fixed enough for submission
- Visible broken text in the project UI was cleaned up.
- Affected project files were also re-saved as UTF-8.

8. CSRF protection is still not implemented
- Status: accepted for submission
- This remains a known low-priority limitation for this local-network project.
