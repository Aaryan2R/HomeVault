1. Project Identity

Project name:
HomeVault

Short description:
HomeVault is a local private cloud storage web application for home/family use. It is inspired by Google Drive and Google Photos, but runs locally on the user's own computer and stores files on local disk.

Core idea:
- upload files from a browser
- store real files on local storage
- store file metadata in SQLite
- show files in a simple web UI
- support search, filtering, download, delete, and photo thumbnails

9. Important Technical Understanding

The project uses two storage layers:

1. Actual file storage
Real uploaded files are stored on disk inside storage/.

2. Metadata storage
File information is stored in SQLite inside homevault.db.

The database stores metadata only, not the actual file content.

Important concepts the user should be able to explain:
- Flask routes handle browser requests
- SQLite stores file metadata
- Jinja renders dynamic HTML
- UUID prevents filename collisions
- Pillow generates photo thumbnails
- Files are categorized by extension
